# eVolleyball

eFootball 型操作の 6人制 3D バレーボールゲーム。
ソロプレイ（vs AI）と 2人マルチプレイ対戦に対応。**PC・スマホ両対応。**

## 遊び方

```
npm install
npm run play      # ビルドしてゲームサーバーを起動（http://localhost:8787）
```

Windows なら **`play.bat` をダブルクリック**するだけで、ビルド→サーバー起動→
ブラウザが開くところまで自動で行われる（常に最新版で遊べる）。

これ1つでゲーム画面と対戦サーバーの両方が同じポート (8787) で動く。
開発時はホットリロード付きの `npm run start`（vite:5173 + サーバー:8787）が便利。

### スマホ2台でオンライン対戦

ゲームサーバーが画面ごと配信するので、スマホ側は **URL を開くだけ**。

**同じ Wi-Fi の場合:**

1. PC で `npm run play` を実行し、PC の IP アドレスを調べる（`ipconfig` の IPv4）
2. スマホ2台のブラウザで `http://<PCのIP>:8787` を開く
   （Windows ファイアウォールの許可ダイアログが出たら許可する）
3. スマホ1が「部屋を作る」→ 左上に出るルームコードをスマホ2に伝える
4. スマホ2がコードを入れて「参加する」

**離れた場所から（インターネット越し）:**

Cloudflare Tunnel で公開するのが簡単（無料・アカウント不要）:

```
winget install Cloudflare.cloudflared
cloudflared tunnel --url http://localhost:8787
```

表示された `https://xxxx.trycloudflare.com` の URL を2台のスマホで開けば、
どこからでも対戦できる（WebSocket は自動的に wss になる）。
※ タイミング判定はサーバー時刻基準のため、遅延が大きい回線では
トスの判定がシビアになる（LAN 推奨）。

相手が来るまでは AI が相手チームを代行し、切断時も AI が引き継ぐ。

## 操作方法（eFootball 型）

選手は全員自動で動く。あなたは「今カーソル ▼ が乗っている選手」のプレーの
質（タイミング・強さ・コース）だけを決める。

| プレー | PC | スマホ |
|---|---|---|
| **サーブ** | SPACE 長押しでパワーゲージ → 離して実行 | アクションボタン長押し→離す |
| **サーブのコース** | ←→ | ◀▶ ボタン |
| **レシーブ** | ボールが届く瞬間に SPACE | 届く瞬間にアクション |
| **トス先の選択** | 1〜4 または矢印キー | 選択肢をタップ |
| **トス** | 到達の瞬間に SPACE | 到達の瞬間にアクション |
| **スパイク** | SPACE 長押しで強さ → 離して確定 / ←→ コース | 長押し→離す / ◀▶ |
| **ブロック** | J/K/L でコース張り + 打つ瞬間に SPACE でジャンプ | タップで張り + 瞬間にアクション |
| カメラ（追従⇔俯瞰） | C | 「視点」ボタン |
| ポーズ / 再戦 / 説明 | P / R / H | ⏸ ボタン / 再戦ボタン |

- パワーゲージはスイートスポット（約7〜8割）があり、溜めすぎるとミスのリスク
- 左下: 操作選手名 + パワーゲージ + レーダー（ミニコート）
- スマホは横向き推奨

## ゲームシステム

- **フル6人制**: 25点ラリーポイント（2点差）、サイドアウトでローテーション、5-1 システム。
  ラリー中は前衛が役割位置へスイッチ（OH=レフト / MB=中央 / OP=ライト）
- **到達点マーカーと軌道線**: ボールの飛行軌道の予測線と、次に落ちる/触られる地点のリングを表示。
  緑=自チームが触る番（セッターはこの印まで移動してトス）、黄=相手ボール・落下点。
  ボールが画面外にあるときは画面端の矢印が方向を示す
- **セッターの自動ポジショニング**: サーブが打たれると自動でネット際の定位置へ走る。
  手動で移動した瞬間に解除される
- **選手ロースター**: 全選手が名前・背番号・身長・能力値
  （攻撃/ブロック/レシーブ/サーブ/速さ）を持ち、試合ごとに生成される。
  能力はスパイク決定率・パス精度・移動速度などに実際に影響する。
  将来の契約・育成機能は `generateRoster()`（src/sim/sim.ts）を差し替える形で載せる
- **一時停止**: ソロプレイ中は P キーまたは右上の ⏸ でポーズ。
  「メニュー」ボタンでいつでもタイトルへ戻れる（マルチ中は AI が引き継ぐ）
- **VAR（ビデオ判定）**: ライン際 35cm 以内の際どい着弾で自動発動。
  地面スレスレのクローズアップカメラにズームし、判定確度%・カウントダウン・
  心拍音の演出のあと IN/OUT が確定する（確定までは得点演出もお預け）
- **ハイライトリプレイ**: スパイク決定/サービスエース/シャットブロックの直後、
  直前 1.4 秒をスローモーション + 回り込みカメラで自動再生（● REPLAY バナー付き）。
  得点チームは歓喜のジャンプ、失点チームはうなだれる
- **トス 5種**: 1=オープン 2=クイック 3=バック 4=二段 5=**ツーアタック**
  （セッターが2本目を流し込む奇襲。相手ブロックがコミットしているほど刺さる）
- **打点マーカー**: スパイク/トスの空中接触点に発光マーカー + 地面への垂直ガイド線
- **慣性移動**: 選手は加速・減速・切り返しに慣性があり、ジャンプ前には踏み込みの
  予備動作が入る
- **試合準備画面**: ソロ開始時に戦術（バランス/攻撃的/堅守）とチーム編成を確認。
  戦術は決定率・ミス率・CPU のブロック傾向に実際に影響する
- **完全手動プレー**: 入力しなければプレーは実行されない。
  サーブ放置=8秒ルール違反 / レシーブ未入力=ノータッチ失点 /
  トス未入力=落球 / スパイク未入力=見送り。あなたが打たなければ誰も打たない
- **リアルな接触とモーション**: サーブはトスアップ→頭上の打点で叩く実シーケンス。
  スパイクは踏み込み→引き腕→振り抜きの3段モーションで、打点はアタッカーの
  身長+ジャンプから計算されボールと手が一致する。全ポーズはブレンディングで滑らか
- **モーションの個性**: 走りのピッチ/腕振り/上下動、踏み切りの深さ、
  スパイクの引き腕の大きさとスイングの鋭さを、能力値+選手固有シードから生成。
  同じ選手はいつも同じ動きをする
- **インパクトIK**: サーブ/スパイクの接触の瞬間、腕がボールの実座標へ向くよう
  補正され、手とボールが必ず接触して見える（腕リーチと打点計算も一致）
- **ギリギリの救出**: タイミングが遅れても（OK/POOR）失敗ではなく、
  腕を目一杯伸ばすランジで何とか拾う。「ギリギリ届いた！」表示
- **タクティカル・ビュー**: セット機会に自動で広角化+カメラ後退し、
  各アタッカーの頭上に選択番号マーカー(1-5)を表示
- **戦術リングメニュー**: Q 長押し（または右上「戦術」ボタン/パッドRB）で
  攻撃指示（オープン/クイック/バック/二段/ツー）をプリセット。トスの初期選択になる
- **ゲームパッド対応**: 右スティックの倒し具合＝スパイク/サーブのコース・
  ブロックの手の出し方（アナログ絶対指定）、A=アクション、十字キー=トス選択/ブロック
- **監督コーチングボード**: V キー（または右上「指示」ボタン）でコートを
  上から見下ろす図解UIを表示。自チームのドットをドラッグして守備位置を
  リアルタイムに微調整できる（試合は進行したまま。リセット可）
- **ヒット感**: スパイク/ブロック接触の瞬間に約85msのヒットストップ、
  ゲームパッド振動（dual-rumble）とスマホバイブで「重み」を伝える
- **連携のビートパルス**: 自チームのレシーブ→トス→スパイクが繋がるたび、
  スコアボードや選手名プレートが青→緑→金の順に鼓動して「リズムに乗っている」
  ことを視覚化
- **トスの質**: タイミング精度＋パス精度（判定窓が変化）＋セッターの位置取りで決まり、
  スパイクの決定率に直結する
- **読み合い**: 相手のセット中にブロックゾーンへ「コミット」できる。
  的中すればシャットブロック率が大幅上昇、外せば逆に決められやすくなる。
  AI もこちらの攻撃傾向（レフト偏重など）を学習してブロックを張ってくる
- **サーブ戦術**: ゾーン指定＋タイミングで、強いサーブほど相手のパスを乱し
  相手セッターのトス判定窓を狭くする

## 検証

```
npm run smoke    # ヘッドレスでシミュレーションを完走テスト（AI戦・疑似人間戦）
npm run build    # 型チェック + プロダクションビルド
node scripts/verify-ui.mjs       # PC ブラウザ自動操作でソロ/マルチの動作確認（要 Edge）
node scripts/verify-mobile.mjs   # スマホ2台エミュレーションでのタップ対戦テスト
```

verify 系は対象サーバーを `BASE` 環境変数で指定できる
（既定: verify-ui は vite:5173、verify-mobile は 8787）。

## アーキテクチャ

```
src/sim/      ゲームシミュレーション本体（three.js 非依存の純ロジック）
              クライアント（ソロ）とサーバー（マルチ）で同一コードを共有
src/client/   three.js 描画・HUD・入力・WebSocket クライアント
src/server/   ルーム制の権威サーバー（60Hz シム / 20Hz スナップショット配信）
```

マルチプレイはサーバー権威型: 入力だけを送り、サーバーのシムが唯一の正。
LAN 前提の設計（タイミング判定はサーバー時刻で行うため、高遅延環境では
トス判定がシビアになる）。
